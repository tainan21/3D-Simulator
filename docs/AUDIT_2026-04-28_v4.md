# Auditoria de 28 de abril de 2026 — release v4

Documento operacional. Lista o que mudou, o que foi preservado, o que ficou para depois e como reverter cada peça em caso de regressão.

## Princípios respeitados

1. A verdade canônica em `src/domain/`, `src/kernel/` e `src/simulation/` não foi tocada.
2. Phaser e Three continuam adapters puros. `canonicalGeometrySignature` mantém a paridade 2D / 2.5D / 3D.
3. Replay determinístico preservado. `FIXED_TIMESTEP = 1/12` continua a única referência de tempo.
4. Toda evolução vive em `infrastructure/`, `app/utils/`, `app/controllers/` ou `runtime/`.

## O que mudou — por fase

### Fase 0 — Baseline e infraestrutura
- Adicionadas dependências `@neondatabase/serverless`, `@vercel/blob` e `idb`.
- `vercel.json` declara `api/**` como Edge Functions com runtime web-standard.
- `scripts/vite/apiDevPlugin.ts` espelha os handlers em `vite dev` (sem `vercel dev`).

### Fase 1 — Banco (Neon + Blob)
- `scripts/db/v1_init.sql` aplicado no Neon (`patient-firefly-13290710`). Seis tabelas, índices e trigger `touch_updated_at`.
- Endpoints Edge: `api/health`, `api/mobs`, `api/archetypes`, `api/runtime`, `api/replays`, `api/settings`.
- Convenção: metadata leve no Postgres, payload pesado no Blob (`mob_blobs`, `runtime_blobs`, `replay_blobs`).

### Fase 2 — Repository + StorageAdapter (cliente)
- `src/infrastructure/repo/{types,createRepository,flushScheduler}.ts` — factory genérica com cache em memória, pub/sub e flush diferido.
- `src/infrastructure/storage/{local,remote,hybrid,select}Adapter.ts` — três modos plugáveis. Default `local` (zero risco).

### Fase 3 — Substituição cirúrgica
- Repositórios concretos: `mob`, `archetype`, `runtimeArtifact`, `replay`, `settings`.
- `src/main.ts` aguarda `initializeRepositories()` antes de montar o shell.
- `src/infrastructure/migration/fromLocalStorage.ts` migra os 5 keys legacy uma única vez (idempotente).
- `src/infrastructure/createdMobCache.ts` virou fachada compatível: callsites do Forge / Studio / Character Studio / Runtime não mudaram, mas agora não tocam storage.
- `src/app/state.ts` reescrito para delegar aos repositórios (API pública preservada).

### Fase 4 — Otimização cirúrgica
- 4.1 — `src/render/dirtyFlags.ts` memoiza chaves por referência via `WeakMap` (zero `JSON.stringify` quando só camera/overlay muda).
- 4.2 — Forge consome `lastSync` do campo + `subscribeCreatedMobCache` para update parcial. `loadCreatedMobCache()` saiu do hot path.
- 4.3 — `captureFocus()`/`restoreFocus()` preservam input/seleção entre re-renders.
- 4.4 — `src/app/utils/{rafThrottle,idleDebounce}.ts` aplicados a pointer/mouse/palette-search/panel-size do Studio.
- 4.5 — `src/app/utils/visibilityPause.ts` (IntersectionObserver) pausa renderers fora da viewport.

### Fase 5 — Forge Animator data-first
- `src/runtime/forgeAnimator.ts` consome `motionProfile.simulationSteps` e publica CSS variables (`--forge-step-x|z|t|progress|phase-index`) sincronizadas com `FIXED_TIMESTEP`.
- Cinco curvas de easing (`start`/`windup`/`travel`/`impact`/`recover`).
- Respeita `prefers-reduced-motion`, `visibilitychange` e `IntersectionObserver`.
- `style.css` ganhou bloco isolado `[data-driven="true"]` que substitui keyframes da `.forge-sim-runner` por posicionamento real, sem desfazer keyframes legados (`.forge-avatar` e variantes intactos).

### Fase 6 — Qualidade
- `tests/unit/forgeAnimator.test.ts`, `tests/unit/repository.test.ts` (mock DOM mínimo, sem nova dep).
- `tests/e2e/forge-flow.spec.ts` cobre Forge → Characters → Runtime persistido.
- Documentos: `docs/PERSISTENCE.md`, `docs/PERFORMANCE.md`, `docs/API.md`.

## Como reverter cada peça em emergência

| Sintoma | Reversão |
|---|---|
| Banco remoto fora do ar | Setar `VITE_STORAGE_MODE=local` no `.env` e redeployar. App volta a funcionar offline-first via IndexedDB. |
| IndexedDB corrompido | Limpar via DevTools → Application → IndexedDB → `rogue-shell-v1`. Migração roda de novo no próximo boot. |
| Forge Animator comportamento estranho | Remover atributo `data-driven="true"` no template do Forge ou comentar o bloco CSS no final do `style.css`. Keyframes legados retomam. |
| Throttle de Studio prejudica autoria | Remover envoltórios `rafThrottle()` em `studioController.ts` (handlers `onPointerMove` e `onMouseMove`). |
| Subscribe do Forge não atualiza | Sintoma de subscriber órfão; checar `cleanups.add` no `mount()`. |

## Como medir o ganho

1. Abrir `/performance` antes e depois — comparar FPS médio em Studio e Forge.
2. Rodar `npm run test` — todos verdes.
3. Rodar `npm run e2e` — `forge-flow.spec.ts` e `screenshots.spec.ts` verdes.
4. Rodar `npm run demarcate` — número de `PARCIAL` não regrediu.
5. DevTools → Memory → snapshot antes e depois de 10min de Studio: GC saudável.
